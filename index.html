AJAX and JSON Demo

This presentation shows how AJAX uses the Fetch API to load JSON data without reloading the page.

What is AJAX?

  • **A**synchronous **J**avaScript **A**nd **X**ML.
  • A set of web development techniques to create more responsive websites.
  • Enables updating parts of a page without a full reload.

Why Use AJAX?

  • **Improved User Experience:** Provides a smoother, more interactive interface without constant page refreshes.
  • **Enhanced Performance:** Websites feel faster as only necessary data is fetched.
  • **Bandwidth Efficiency:** Reduces server load by only fetching needed content.
  • **Desktop-like Functionality:** Web apps can behave more like traditional desktop applications.

How Does AJAX Work?

  1. **User Action:** An event (e.g., button click) triggers JavaScript.
  2. **Request Object:** JavaScript uses the `XMLHttpRequest` object (or `Fetch` API) to send a request to the server in the background.
  3. **Server Processing:** The server processes the request and retrieves data.
  4. **Server Response:** The server sends data back (often as JSON).
  5. **DOM Update:** JavaScript updates only the relevant part of the webpage using the Document Object Model (DOM) without reloading the entire page.

The Code Example

We use the **Fetch API** to get data from a file called users.json.


fetch('users.json')
    .then(response => response.json())
    .then(users => {
        // Code to display users
    });

This is all done behind the scenes, making the user experience seamless.
